# 07 · The Museum Expansion — From Exhibit Hall to Institution

This pass turns DECRYPTED from "a few exhibits in a room" into a **dense, grand,
world-class museum of cryptology** — without hand-editing a single line of scene
YAML. In keeping with the rest of the project, the new content is delivered **as
code**: a small editor toolkit that *constructs* the museum procedurally, plus a
fully-authored curatorial script. Run one menu command and all six galleries are
built, dressed and lit.

> TL;DR — open the scene, run **`DECRYPTED ▸ Museum ▸ Build Full Museum`**
> (`Ctrl/Cmd + Shift + M`), then press Play. Re-run any time; it is idempotent.

---

## Why a builder instead of a saved scene?

The original `Decrypted_Main.unity` is, in practice, still close to the
**skeleton** that `SceneBuilder` stamps out — six room roots, each with a player
anchor and a floor plane, the XR rig and the manager rig. There were no walls,
ceilings, exhibits, plaques or signage placed in the scene. That is exactly why
it read as empty.

Rather than freeze hundreds of objects into an opaque 8,000-line scene file, the
expansion follows the project's existing philosophy (see `03_Asset_Production_Pipeline.md`):
**assets as reviewable, reproducible code.** The museum is generated by editor
scripts from authored data, so the entire build is a *diff*, not a binary blob,
and a curator can proof-read every word in one file.

---

## What gets built

For **every** gallery the builder produces:

- **A grand architectural shell** — a 15–17 m hall with a 6–7 m ceiling, a
  coffered beam grid and a glowing skylight, a four-column colonnade down each
  long wall, framed-archway doorways, baseboard + cornice trim, a carpet runner
  with a trim border, and emissive wall sconces.
- **A dramatic hero centerpiece** on a plinth with a rope barrier, benches and
  real spotlights — a turning Caesar disk, an Enigma machine, a circular vault
  door, a morphing synthesis sculpture, a rotating armillary globe, or a glowing
  entrance portal.
- **Dense perimeter exhibits** drawn from the content database — glass display
  cases, spotlit pedestals (with slowly-turning artifacts), angled tablet cases
  for manuscripts and maps, and wall-mounted reliefs — **each with its own
  museum label** (Title / Date / Description / Significance).
- **A portrait gallery** of the pioneers of the field, with engraved nameplates
  and biographies that fade in as you approach.
- **A wall timeline** of that era, **framed infographic panels**, an
  **interactive guide kiosk** (or a staffed **information desk** in the lobby and
  rotunda).
- **The realism details** — hanging gallery signs, a directory, a back-wall
  gallery sign, exit/return signage over the doorways, velvet banners, planters,
  a trash bin, a donation box, security cameras, and a floor medallion.

Only **one room is ever active at a time** (the project's room-based culling), so
the per-frame cost stays close to a single dressed gallery.

## The six galleries (a guided arc: Past → Present → Future)

| Gallery | Identity | Hero | A few of its artifacts |
|--------|----------|------|------------------------|
| Splash | **Welcome Pavilion** | Glowing entrance portal + floor seal | Visitor guide, mission wall |
| Atrium | **Orientation Rotunda** | Rotating armillary globe | Grand 3,000-year timeline, Kerckhoffs's Principle, the CIA triad |
| Ancient | **Gallery of Antiquity** | The Caesar cipher disk | Scytale, Atbash, Polybius square, Al-Kindi, Alberti, Vigenère, Babington plot, the Great Cipher, the Voynich manuscript |
| WWII | **Hall of the World Wars** | The Enigma machine | Zimmermann Telegram, the one-time pad, Rejewski, the Bombe, Bletchley, Lorenz & Colossus, the Navajo Code Talkers, PURPLE, SIGABA |
| Vault | **The Digital Age** | The security vault | Shannon, DES, Diffie–Hellman, RSA, public-key, AES, hashing, digital signatures, TLS, the Crypto Wars |
| Reveal | **The Quantum Future** | The synthesis sculpture | Quantum key distribution, Shor's algorithm, post-quantum cryptography, the right to privacy |

Every label is original, historically grounded prose. There is **no placeholder
text** anywhere in `MuseumContent.cs`.

## The toolkit (where to look / how to tweak)

```
Assets/_Project/Scripts/Editor/Museum/
├── MuseumContent.cs   The curatorial script — galleries, ~50 exhibits, figures,
│                      timelines, signage copy. Edit text here.
├── MuseumKit.cs       URP material palette (shared .mat assets) + primitive,
│                      text and light builders. Tune the palette here.
├── MuseumProps.cs     Display cases, pedestals, plaques, columns, benches,
│                      kiosks, signage, decor and the procedural artifacts.
└── MuseumBuilder.cs   The orchestrator + DECRYPTED ▸ Museum menu. Per-gallery
                       sizes, styles, hero pieces and layout live here.

Assets/_Project/Scripts/Visuals/MuseumAmbience.cs
   MuseumSpin / MuseumGlowPulse — tiny runtime "alive" helpers (turntables,
   breathing display lights). Allocation-free; safe to remove.
```

Generated materials are written once to
`Assets/_Project/Art/Generated/Museum/Materials/` and reused on every rebuild
(so they batch under the SRP Batcher).

### Menu commands (`DECRYPTED ▸ Museum`)

| Command | Effect |
|---------|--------|
| **Build Full Museum** (`Ctrl/Cmd+Shift+M`) | Builds/refreshes all six galleries. Idempotent. |
| **Clear Generated Dressing** | Removes everything the builder added (the `MuseumDressing` child of each room). |
| **Rebuild One ▸ Selected Room** | Select a `Room_*` object and rebuild just that gallery — fast iteration. |

Everything generated lives under a single **`MuseumDressing`** child per room, so
nothing here touches the managers, the XR rig, the save system or the wired
interactive exhibits. Each hero also leaves a clearly-named
`HERO_ANCHOR_DropInteractiveExhibitHere` empty so the fully-wired Caesar/Enigma/
Vault prefabs can be parented onto the centerpiece when you are ready.

## How to run it

1. Open the project in **Unity 2022.3 LTS** and open
   `Assets/_Project/Scenes/Decrypted_Main.unity`.
2. Run **`DECRYPTED ▸ Museum ▸ Build Full Museum`** (or `Ctrl/Cmd+Shift+M`).
   The Console prints how many galleries were built.
3. To preview a gallery in the editor, select its `Room_*` root and tick it
   active in the Inspector (only Splash starts active at runtime).
4. **Press Play** — Demo Mode walks the rooms; or build to the Quest.

### Final-polish (optional)

- **Bake lighting** for the richest look: `Window ▸ Rendering ▸ Lighting ▸
  Generate Lighting`. The builder's lights are real-time so the scene looks
  correct immediately, but a bake adds soft shadows and bounce and frees the
  runtime budget (set the added lights to *Mixed/Baked* before baking if you
  target standalone Quest).
- The emissive "display lights", sconces, status lamps and screens cost **no**
  light at all, so most of the glow survives even with lighting disabled.

## Design notes

- **Density from every angle.** Heroes sit forward of the player spawn so you
  face them on entry; cases ring the perimeter; walls carry panels, portraits and
  a timeline; the ceiling carries beams, a skylight and banners; the floor carries
  a runner, a border and a medallion. A 360° turn always lands on content.
- **Legibility in VR.** Plaque *titles* and signage are always visible (the room
  reads as full from across the hall); long *body* copy fades in within ~3 m via
  the existing `PlaqueController`, which also saves Quest fill-rate.
- **One palette, six moods.** Warm torch-lit sandstone for Antiquity, cool
  instrument grey for the World Wars, clean steel-and-cyan for the Digital Age,
  theatrical gold-on-black for the Reveal — colour temperature tells the era,
  exactly as `02_Environment_Design.md` intends.
